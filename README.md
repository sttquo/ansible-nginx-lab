# Автоматизация инфраструктуры на Ред ОС с помощью Ansible

Репозиторий содержит комплекс сценариев для автоматической подготовки серверов и развертывания веб-сервера Nginx на хостах под управлением Ред ОС. Проект демонстрирует разделение логики плейбуков и конфиденциальных данных (секреты вынесены в переменные).

---

## 🖥️ Архитектура стенда (VirtualBox)

Стенд развернут локально в Oracle VirtualBox и состоит из трех виртуальных машин:
* **`master-node`** (Управляющий узел) — машина, с которой запускается Ansible.
* **`10.0.2.22`** и **`10.0.2.23`** (Целевые узлы) — серверы на Ред ОС.

**Настройка сети:** все машины находятся в одной подсети (тип подключения в VirtualBox — NAT с пробросом портов). Между управляющим и целевыми узлами настроен беспарольный доступ по SSH-ключам (`ed25519`).

---

## 📂 Структура проекта и сценарии

* `hosts` — инвентарь с IP-адресами целевых серверов.
* `ansible.cfg` — конфигурация с отключенной проверкой хост-ключей (`host_key_checking = False`) для удобства в лабораторной среде.
* `group_vars/red_os_servers.yml` — общие переменные группы хостов. Сюда вынесены пути к ключам, версия Python (3.11) и захешированный пароль для нового пользователя, чтобы не светить конфиденциальные данные в коде плейбуков.

### Плейбуки:
1. **`setup_env.yml`** — Подготовка окружения. Создает на целевых серверах пользователя `labuser`, выделяет ему директорию `/opt/lab_folder` с нужными правами и генерирует приветственный текстовый документ.
2. **`nginx_install.yml`** — Развертывание веб-сервера. Обновляет кэш пакетов, устанавливает `nginx` через пакетный менеджер `dnf`, запускает службу Nginx и добавляет её в автозагрузку системы.

## 🚀 Инструкция по запуску

1. Включите виртуальные машины в VirtualBox.
2. Проверьте доступность узлов и корректность настройки SSH-доступа:
   ```bash
   ansible red_os_servers -m ping

## Как это запускать (памятка для себя)

1. Включаем виртуалки в VirtualBox.
2. Проверяем, что Ansible их видит:
   ```bash
   ansible red_os_servers -m ping```

3. Запускаем базовую настройку окружения (создание пользователя и директорий):
   ```bach
